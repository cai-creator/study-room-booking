# 研习室预约系统 - 使用指南

## 快速开始

### 1. 启动 MySQL 数据库

项目已内置 MySQL 9.7，位于 `mysql/mysql-9.7.1-winx64/`。

```bash
# 启动 MySQL
mysql\mysql-9.7.1-winx64\bin\mysqld.exe --console

# 验证 MySQL 是否启动
mysql\mysql-9.7.1-winx64\bin\mysqladmin.exe -u root -proot ping
```

### 2. 初始化数据库（仅首次）

```bash
cat docs/sql/schema.sql | mysql\mysql-9.7.1-winx64\bin\mysql.exe -u root -proot
```

> 已初始化可跳过。脚本会重建所有表并插入示例数据（校区、楼栋、座位等）。

### 3. 启动后端

```bash
# 开发环境（连接 MySQL，默认）
mvn spring-boot:run

# 或使用 H2 内存数据库（无需 MySQL）
mvn spring-boot:run -Dspring-boot.run.profiles=h2
```

后端启动在 **8081** 端口，API 前缀 `/api`。

### 4. 启动前端

```bash
cd frontend
npx serve .
# 或
python -m http.server 3000
```

访问 `http://localhost:3000/public/login.html`

## 常见问题

**Q: 前端登录提示 "网络错误"？**
A: 确保后端已启动在 8081 端口。检查 `config.js` 中 `apiBaseUrl` 是否指向正确的后端地址。

**Q: 登录提示 "用户名或密码错误"？**
A: 确认使用正确的测试账号。如果连的是真实 MySQL，确认已执行过 `docs/sql/schema.sql` 初始化。

**Q: CORS 跨域错误？**
A: 后端已配置 `allowedOriginPatterns("*")`，支持 `file://` 和 `localhost` 访问。

**Q: 如何切换 Mock 模式？**
A: URL 加 `?mock=true` 使用 Mock，不加则连真实 API。
